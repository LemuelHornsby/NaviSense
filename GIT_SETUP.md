# Git setup — off-machine backup for the NaviSense UE5 workspace (run on Windows)

The sandbox can't manage git on the `D:` drive, so do this once in **PowerShell on your PC**.
Goal (KI-006): a full off-machine backup of THIS workspace on your existing remote
**https://github.com/LemuelHornsby/NaviSense** — replacing its dated history with a clean snapshot.

**Scope (decided 2026-07-14):** back up **code + docs + UE Content + ship models** (~3 GB, via Git LFS —
fits GitHub's free **10 GiB** LFS storage). The **2.4 GB demo films** (`Demo/film/`) are **excluded** —
they're large, regenerable, and already on your HDD (put them on a cloud drive if you want them off-site;
LFS is a poor fit for video). Transient UE dirs (Saved/Intermediate/Binaries/DerivedDataCache/.vs),
the 832 MB Cesium tile cache, `logs/`, and `.venv/` are git-ignored and must never be committed.

---

## 0 · One-time prerequisites

```powershell
# Git for Windows already includes Git LFS; enable it once per machine:
git lfs install

# UE asset paths are long — allow them, and set your identity if not already set:
git config --global core.longpaths true
git config --global user.name  "Lemuel Hornsby"
git config --global user.email "navisynmarinesolutions@gmail.com"

# Authenticate to GitHub (simplest: GitHub CLI). Either:
#   winget install GitHub.cli   ; gh auth login          (browser login)
# or let Git Credential Manager prompt for a Personal Access Token on first push.
```

## 1 · Create the repo and stage everything

```powershell
cd "D:\Marine Autonomy\NAVISENSE\NaviSense Simulator with Unreal Engine"

git init -b main                 # fresh repo (there is no .git here yet)

# Commit the LFS + ignore rules FIRST so the filters apply to everything added next:
git add .gitattributes .gitignore
git commit -m "chore: Git LFS + ignore config"

# Stage the whole workspace — .gitignore drops Saved/Intermediate/Binaries/.vs/logs/
# the Cesium cache/.venv and Demo/film automatically:
git add -A
```

## 2 · PRE-PUSH AUDIT (do NOT skip — this is what makes the push succeed)

GitHub hard-rejects any **non-LFS** file over **100 MB**. This check must print **nothing but LFS=True**:

```powershell
# Every tracked file > 95 MB and whether Git LFS owns it:
git ls-files | ForEach-Object {
  $it = Get-Item -LiteralPath $_ -ErrorAction SilentlyContinue
  if ($it -and $it.Length -gt 95MB) {
    $isLfs = (git check-attr filter -- $_) -match 'lfs'
    "{0,7:N0} MB  LFS={1}  {2}" -f ($it.Length/1MB), $isLfs, $_
  }
}
```
- **Every line must show `LFS=True`** (expected big ones: `marine_rescue_boat.uasset` ~494 MB,
  `excursion_vessel.uasset` ~354 MB, `dolphin.3dm` ~339 MB, `hull.uasset` ~114 MB, `hull.fbx` ~107 MB).
- If any line shows `LFS=False`, STOP: add that extension to `.gitattributes` as an `filter=lfs …` line, then
  `git rm --cached <file>` → `git add .gitattributes` → `git add <file>` and re-run the audit.

Confirm LFS captured the asset set (expect several hundred objects):
```powershell
git lfs ls-files | Measure-Object          # ~600+ objects (uasset/umap/fbx/3dm/png/…)
git status --short | Measure-Object        # sanity: how many paths staged
```

## 3 · Commit and push (replaces the dated remote history)

```powershell
git commit -m "NaviSense UE5 workspace — authoritative snapshot 2026-07-14 (code, docs, UE Content + assets via LFS)"

git remote remove origin 2>$null
git remote add origin https://github.com/LemuelHornsby/NaviSense.git

# --force REPLACES the old sparse history with this clean snapshot (your choice 2026-07-14):
git push -u origin main --force
```
The first push uploads ~3 GB of LFS objects — a few minutes on a normal connection. Uploads count
against **storage** (~3 of 10 GiB), not bandwidth.

## 4 · Verify + tidy on GitHub

```powershell
git lfs ls-files | Measure-Object          # objects now on the remote
```
- Open **https://github.com/LemuelHornsby/NaviSense** — files show; large assets display a
  **"Stored with Git LFS"** badge; `NaviSense_UE5/Content/…` is present.
- **Settings → General → Default branch → `main`**, then delete a stale `master` branch if one remains
  (Branches page). Keep the repo **Private**.

## 5 · Ongoing backup (after each work session / before closing UE)

```powershell
cd "D:\Marine Autonomy\NAVISENSE\NaviSense Simulator with Unreal Engine"
git add -A
git commit -m "snapshot: <what changed>"
git push
```
Re-run the §2 audit only if you add a **new kind** of large binary. Tip: back up right after an
in-editor session so Blueprint/level/data-asset edits (which live in `Content/*.uasset`) are captured.

## 6 · What's intentionally NOT in git (and where it lives)

| Excluded | Size | Why / where it lives |
|---|---|---|
| `NaviSense_UE5/Saved`, `Intermediate`, `Binaries`, `DerivedDataCache`, `.vs` | ~9 GB | Regenerated by UE/VS on build — never version these |
| `**/cesium-request-cache.sqlite*` | ~1.2 GB | Machine-local Cesium tile cache — re-downloads on demand |
| `Demo/film/` | ~2.4 GB | Large regenerable demo videos — HDD backup (+ a cloud drive if you want them off-site) |
| `logs/`, `Saved/NaviSense_Reports/` | ~0.4 GB | Run outputs — regenerated by `run_demo.py` |
| `*.3dmbak`, `.venv/`, `__pycache__/` | — | Rhino auto-backup / local Python venv / caches |

If a future asset type gets rejected by the push, it just needs a `filter=lfs` line in `.gitattributes`.
