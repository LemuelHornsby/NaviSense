## 1 · QA principles

- **The log is the source of truth**, not the screen. Acceptance is decided from `state.csv`/`sensor.csv`
  and analysis outputs, with visuals as confirmation.
- **Validated, not just pretty:** maneuver behavior is checked against IMO criteria and (ultimately) the
  CFD/tow-tank numbers — that is the product's differentiator.
- **Automate verification, keep judgment human.** Every test writes a machine-readable result where it can.
- **No gate is "passed" without evidence** (a result file, a log, or a dated tester sign-off).

## 2 · Test levels

| Level | What it proves | Where |
|---|---|---|
| **L0 Protocol (offline)** | The bridge wire contract round-trips without Unreal | `Development/bridge_harness/` |
| **L1 Smoke (in-engine)** | Module compiled; pawn connects; loop runs | preflight script + a short PIE run |
| **L2 Correctness** | Coordinate/sign convention is right | sign test (harness + in-engine) |
| **L3 Maneuver KPIs** | Dynamics match IMO maneuver criteria | `python/analyse_*.py` on a logged run |
| **L4 Regression** | Nothing previously working broke | the regression checklist (§5) |
| **L5 Acceptance** | A demo/release gate is met | demo gates D1–D8 (§6) |

## 4 · Per-Work-Packet gates

Every Work Packet (`Development/work_packets/WP_*/PACKET.md`) defines its own acceptance gates and ships a
`verify_*.py` that writes `Saved/NaviSense_Reports/wp_*_result.json`. A packet is **closed only when all its
gates pass** (automated result file + any manual PIE confirmation). Failed gates roll into the next packet —
never silently dropped. This is the unit-level QA loop.

## 7 · Severity & exit criteria

- **S1 Blocker** — no demo possible (loop dead, crash, wrong sign). **Release exit:** zero open S1.
- **S2 Major** — a core feature broken with no workaround. **Release exit:** zero open S2 for the demoed path.
- **S3 Minor** — workaround exists. **Release exit:** documented in Known Issues.
- **S4 Cosmetic** — log and schedule.
