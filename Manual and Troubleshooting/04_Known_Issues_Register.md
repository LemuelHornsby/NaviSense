**KI-001 / KI-011 (the "straight line, no zig-zag" investigation).** The 14 Jun 20:57 run logged
`mode=approach`, `rudderCmdDeg=0`, `yawDeg=0` for all 7,620 rows — the ship accelerated straight and never
maneuvered. Root cause: the `ZigzagController` only begins after the 60 s approach *and* once it can read
heading from the inbound sensor packet; the pawn's `Bridge->SensorBundle` was unset, so packets carried no
IMU heading, so the controller sat in approach forever. Two-part fix (14 Jun): (1) listener now injects the
plant's authoritative yaw into the controller view; (2) the pawn auto-wires its `SensorBundle`. Verified
headless with a client sending *no* IMU — rudder reverses, ship turns.
